Understand where things live in the codebase and what each folder provides.

## At a glance
- `src/` React app (components, pages, utils, styles, tests)
- `public/` static assets (wiki source, resources, fonts, bible data)
- `supabase/` database migrations
- `docs/` Vite build output for GitHub Pages (generated)
- `scripts/` maintenance utilities (index build, normalization, wiki sync)

### `src/`
SPA entry point and components. Routes include Home, SongView, Setlist builder, Songbook builder, Resources, Worship Mode, Editor Portal, and Admin Portal.

Key areas:
- `src/pages/AdminPage.jsx` — Admin Portal (user table, role management, collaborator request queue); requires `admin` role
- `src/pages/EditorPage.jsx` — Editor Portal landing page; requires `editor` role
- `src/components/auth/RoleGuard.jsx` — route guard that redirects users lacking the required role
- `src/components/CollaboratorRequest.jsx` — request button for `user`-role accounts wanting collaborator access
- `src/hooks/useAuth.jsx` — auth context with full role system (`role`, `hasMinRole`, `isOwner`, `isAdmin`, etc.)
- `src/lib/supabase.js` — Supabase client initialization
- `src/utils/setlists/supabaseSets.js` — Supabase-backed saved set operations
- `src/utils/pdf_mvp/` — single-song PDF engine with tests and font registrar
- `src/utils/pdf/` — facade and multi-song/songbook exports
- `src/utils/chordpro/` — parser, serializer, normalization, and helpers
- `src/styles/tokens.css` — UIKit-inspired design tokens (light/dark, spacing, type)
- `src/styles/admin-portal.css` — Admin/Editor portal styles
- `src/components/ui/layout-kit/` — reusable UI primitives (Card, Toolbar, Chip, etc.)
- `src/features/readings/` — Daily Word reading view (M'Cheyne plan + translation-aware Bible loader)

### `public/`
Contains `resources/` for blog posts, `wiki/` as the source of these pages, and `bible/` for translation manifest + generated chapter JSON. UI fonts live under `public/fonts/`; PDF export fonts are bundled under `src/assets/fonts/`. Songs are stored in Supabase, not in `public/`.

### `supabase/`
SQL migration files:
- `migrations/20240001_create_user_starred_songs.sql` — starred songs
- `migrations/20240002_fix_starred_songs_fk.sql` — FK fix
- `migrations/20240003_add_delete_user_function.sql` — account deletion RPC
- `migrations/20260305_songs_migration.sql` — songs table, RLS policies, star-count trigger
- `migrations/20260307_collaborator_requests.sql` — collaborator access request queue

### `docs/`
Generated by `npm run build` as the GH Pages root. Do not edit by hand.

### `scripts/`
Utility scripts: `buildIndex.mjs`, `buildResourcesIndex.mjs`, `normalizeFilenames.mjs`, and `syncWiki.mjs`.

`scripts/ingest/` contains the standalone ingestion CLI (`gc-ingest`) plus local inbox/staging:
- `scripts/ingest/_ingest_inbox/` — drop source PDFs/DOCX/images here
- `scripts/ingest/_ingest_staging/` — generated drafts, normalized outputs, and reports
